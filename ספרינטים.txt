🧭 Sprint 1 – MVP Frontend (ממשק משתמש בסיסי)

מטרה: ממשק עובד שמאפשר לבחור פילטרים ולדמות יצירת דבר תורה.

משימות:

יצירת פרויקט React (עם TypeScript או JS רגיל).

התקנת Bootstrap והגדרת עיצוב בסיסי.

בניית קומפוננטות:

FiltersPanel

GenerateButton

Loader

ResultSection

HomePage

חיבור לוגיקה בסיסית בין הקומפוננטות (state, props).

סימולציה של תהליך יצירה (setTimeout שמחזיר לינק לקובץ דמה).

עיצוב כולל, שמירה על UI אחיד ואסתטי.

תוצר: אפליקציית Frontend מלאה, עובדת מקומית, בלי Back-End אמיתי עדיין.

⚙️ Sprint 2 – Backend בסיסי ו־API חיצוני

מטרה: לבנות API שמחזיר קובץ לדוגמה ולחבר אותו ל־Frontend.

משימות:

יצירת פרויקט ASP.NET Core Web API.

יצירת Controller בשם TorahController עם endpoint:
POST /api/torah/generate שמחזיר קובץ DOCX/PDF קבוע (dummy file).

הגדרת CORS כך שה־Frontend יכול לשלוח בקשות.

עדכון React לקרוא ל־API האמיתי בעזרת fetch.

בדיקה שהתוצאה שמורה בלינק הורדה אמיתי.

תוצר: מערכת מלאה שמדמה יצירת קובץ אמיתי דרך שרת.

🧠 Sprint 3 – אינטגרציה עם OpenAI (תוכן אמיתי)

מטרה: להפוך את יצירת הקובץ מדינמית ואמיתית בעזרת מודל שפה.

משימות:

יצירת TorahGeneratorService ב־Backend.

חיבור ל־OpenAI API (או Mock אם אין גישה זמנית).

העברת הפילטרים מ־Frontend ל־Backend.

יצירת טקסט בהתאם לפילטרים הנבחרים.

החזרת טקסט ללקוח (בשלב זה אפשר רק טקסט, בלי קובץ).

תוצר: מערכת שמחזירה דבר תורה אמיתי שנוצר על פי בחירות המשתמש.

📄 Sprint 4 – הפקת קובץ Word/PDF מעוצב

מטרה: להמיר את התוכן שנוצר לקובץ Word או PDF עם עיצוב תורני נקי.

משימות:

יצירת FileService שאחראי להפקת DOCX או PDF.

עיצוב אחיד: כותרת, מחבר, פרשה, גוף דבר התורה.

שמירת הקובץ באופן זמני והחזרתו להורדה למשתמש.

בדיקות תקינות: גודל קובץ, שמות קבצים, תמיכה בעברית.

תוצר: המשתמש מקבל קובץ מעוצב ואיכותי — "מוצר גמור".


🧩 Sprint 5 – לוגיקה כבדה בפרונט - כפתורים, והצגת הדבר תורה כטקסט מעוצב.

🧩 Sprint 6 – הרחבות ומערכת ניהול

מטרה: להתחיל לחשוב קדימה על פיצ'רים מתקדמים.

משימות אפשריות:
